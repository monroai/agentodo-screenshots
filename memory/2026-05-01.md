### Lessons Learned (from Oleksandr's feedback)
1. **Always attach screenshots** — multiple issues got "attach screenshot" comments. No excuses.
2. **Verify which app you're looking at** — filed #241 about Claude's sidebar thinking it was AgenToDo. Embarrassing.
3. **Know standard UI patterns** — #218 was just a current time indicator in calendar view. Basic stuff.
4. **Don't assume setup context** — #234, onboarding wasn't bugged, data was pre-loaded intentionally.
5. **"Different" ≠ "wrong"** — #243 pagination style was intentional design. Read the design language before applying HIG dogmatically.
6. **Question the premise first** — #229 was filed as "missing subtitle" when the right take was "subtitles are clutter, remove them." Think like Jobs: simplify, don't add.
7. **File issues as you go** — don't batch. Session can hang, context can be lost.

### Rules for Future Reviews
- Crop and attach a screenshot to EVERY issue
- File each issue immediately upon discovery
- Triple-check which window/app is in frame
- Default stance: "should this exist?" before "this is missing"
- Know platform conventions (time indicators, standard controls) before flagging
- When in doubt, propose simplification over addition

### Screenshot Workflow for Issues
1. Capture relevant screenshot with `screencapture -o -x <path>`
2. Crop to highlight the bug area with `sips --cropToHeightWidth H W --cropOffset Y X`
3. Copy to `/tmp/agentodo-screenshots/` and `git add . && git commit -m "desc" && git push`
4. Reference in issue body: `![desc](https://raw.githubusercontent.com/monroai/agentodo-screenshots/main/<filename>.png)`
5. **EVERY issue gets a screenshot** — no exceptions. If you can't screenshot it, describe the exact visual with a code reference.

### Think Like Steve Jobs — Issue Quality Bar
- **The product IS the experience.** Don't file "the color is slightly off" — file "this misleads the user because..."
- **Root cause > symptom.** Always dig into the code. Attach the file + line number.
- **One issue = one clear fix.** If the fix isn't obvious from the report, rewrite it.
- **Would this ship at Apple?** That's the bar. Not "is this technically broken" but "would a user feel delighted?"
- **Simplify, don't add.** Default to removing clutter, not adding features. The calendar banner doesn't need a dismiss button — it needs to not exist in the main view.
- **Cross-platform = one product.** If iOS and macOS show different numbers for the same data, the product is broken. Period.
